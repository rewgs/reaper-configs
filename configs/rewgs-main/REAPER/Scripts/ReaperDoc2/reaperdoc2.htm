REAPERDOC2

Summary

Welcome to ReaperDoc2 - a tool to convert Reaper project files into a readable printable format.  Template files are used to determine how to format the output file.  Written in Python. This means that it can run on both Windows and Mac and can be used as a ReaScript within Reaper. It also can run standalone like the original version.

The original ReaperDoc was developed by Gregg Reno http://www.greggreno.com using Visual Basic. He developed the algorithms and templates and original documentation and sample output used in this product and all thanks and credit for the ideas and work should go to him.


You can discuss the development of this program in The Cockos Forums.

NOTE:  If you are viewing this page from within ReaperDoc2, you can always find this htm file in your ReaperDoc2 installation directory.

Installation

Although Python scripts (even ReaScripts) can be installed and run from any place on your harddrive, I recommend that all scripts intended to be ReaScripts be installed in the REAPER/Scripts application data directory. You can find this location by in Reaper from the main menu by clicking "Options -> Show REAPER Resource Path in Explorer/Finder". I have found that there might not be a Scripts folder under the REAPER folder. If the Scripts folder does not exist, you should create one.

Python must be installed. Usually Mac OS/X already has Python installed. This link http://www.python.org/download/ lists the versions of Python available.

To run ReaperDoc2 as a custom action within Reaper, ReaScripts must be enabled in Reaper. Click "Options -> Preferences -> Plug-ins -> ReaScript". Check the Enable box. On that dialog also check that it has correctly identified Python.

Simply unzip the entire contents of the ReaperDoc2.zip (including folder hierarchy) file into the Scripts directory. The file reaperdoc2.htm is the file you are reading now. It is also available from the help menu of ReaperDoc2.

To make this a custom action in Reaper (a ReaScript), click "Actions -> Action List" in the main menu. Then, click the ReaScript: New/Load button. This should take you to Scripts folder. Navigate to the Scripts/ReaperDoc2 folder and select reaperdoc2.py. Now you have a new action in your action list.

Preferences

The preferences are available from the "File -> Preferences" menu item. These selections are remembered by ReaperDoc2. You don't have to select them every time you run the program.

  • Separator: This is the separator that will be used between Items, FX, Sends, Receives, and other things and other things that occur in multiples on a track.
    1. "<BR>", the HTML line break. Aways use this for HTML output, if you want line breaks between list items. This is the default.
    2. "comma", places a comma between list items.
    3. "cr", places a single character carriage return (\r or chr(13)) between list items. May be useful in some instances. Ignored by HTML.
    4. "linefeed", places a linefeed character (\n or chr(10)) between list items. Useful for csv files when you want the lists of FX, Sends, Receives to display in a single cell on different lines in your spreadsheet. Might also be useful if the output file will be used in Linux. Ignored by HTML.
    5. "crlf", carriage return/linefeed which is the standard Windows line end (\r\n). Might be used like linefeed above for Excel, but I find unreliable. Otherwise just for text formating. Ignored by HTML.
    6. "space", a simple ascii space to be used for separating items.
    7. "tab", may be used for tab-delimited files or for simple formatting in text files. Ignored by HTML.
  • Date Format: A variety of date formats attempting to allow for various country's preferences. Defaults to "yyyy-mm-dd".
  • Time Format: A variety of time formats attempting to allow for various country's preferences. Defaults to "hh:mm:ss", 24 hour.

Using ReaperDoc2

It's pretty easy to use ReaperDoc2:

  • The Reaper Project must be saved.
  • To run standalone, just double-click on reaperdoc2.py.
  • To run as a Reaper custom action, see Installation.
  • Set your preferences if they need to be changed. See Preferences.
  • Select three files in the dialog box:
    • Select a Reaper project file to open, in ReaScript mode the Browse button will go to the project directory.
    • Select an output HTML file name, in ReaScript mode the Browse button will go to the project directory.
    • Select a template to use.  This will be pre-filled with the default template. The browse button will take you to the template directory.
  • Click the "Create File" button
  • ReaperDoc2 will write to the HTML file you selected using the current template file (see Templates section for more info)
  • The HTML file will open in the program's browser window.
  • If you want, you can use the Browser's File -> Print menu item to print the document

Templates

ReaperDoc uses a file called a Template to determine the format of the output file.  A template is simply a formatted file (usually HTML) with some special keywords embedded within it.  You can use the included templates, or create your own.

When converting a project file, the keywords will be replaced with data from the project file.  Keywords all start and end with double pound characters (##) .  The current list of keywords are:
 

Project Level Info
##FILE-NAME## The name of the Reaper project file
##VERSION## Reaper format version number
##TEMPO## Song tempo
##SAMPLE-RATE## Sample Rate
##NOTES## Project notes / comments

Track Level Info
##TRACK-NUM## Track Number
##TRACK-NAME## Track Name
##VOLUME## Track Volume
##PAN## Track Pan
##MUTE## Track Muted.  "Mute" if muted, otherwise blank
##SOLO## Track Solo.  "Solo" if track solo is on, otherwise blank
##FX-LIST## Simple list of effects - no special group tags needed
##ITEM-LIST## Simple list of items - no special group tags needed
##SEND-LIST## Simple list of sends - no special group tags needed
##RCV-LIST## Simple list of receives - no special group tags needed

Effects Level Info
##FX-NUM## Effects sequence number
##FX-NAME## Effects name
##FX-TYPE## Effects type (ie VST, DX, etc)
##FX-DLL## Effects DLL (used for VSTs)
##FX-ACTIVE## Effects Active? ("Active" if active, otherwise blank)

Send Info
##SEND-NUM## Send sequence number
##SEND-TRACK-NUM## Send To track number
##SEND-TRACK-NAME## Send To track name
##SEND-MUTE## Send To Muted.  "MUTE" if muted, otherwise blank
##SEND-VOLUME## Send To Volume
##SEND-PAN## Send To Pan

Receive Info
##RCV-NUM## Receive From sequence number
##RCV-TRACK-NUM## Receive From track number
##RCV-TRACK-NAME## Receive From track name
##RCV-MUTE## Send To Muted.  "MUTE" if muted, otherwise blank
##RCV-VOLUME## Send To Volume
##RCV-PAN## Send To Pan

Item Level Info
##ITEM-NUM## Item sequence number
##ITEM-NAME## Item Name (sound file name)
##ITEM-POS## Item Position
##ITEM-LEN## Item Length
##ITEM-GROUP## Item Group Number
##ITEM-MUTE## Item Muted.  "MUTE" if muted, otherwise blank
##ITEM-VOLUME## Item Volume
##ITEM-PAN## Item Pan

If / Then / Else
<#IF Beginning of If Then Else structure.  <#IF must be followed by one of the following:
  • -IN-GROUP - If track is inside a group
  • -IS-GROUP - if this track is the group header
  • -IS-MUTED - if track is muted

The keyword is followed by user defined tags or text which will only be output if the condition is met.

Note that there cannot be a space between the ##IF and the condition. For example, "<#IF-IN-GROUP" is valid but "<#IF -IN-GROUP" is not.
##ELSE Tags or text following the ##ELSE statement will be output if the condition is NOT met
##END-IF#> Ends the If Then Else structure

Other Info
##TEMPLATE-NAME## The name of the template used for the conversion
##OUTPUT-NAME## The name of the HTML output file
##DATE## Current date - uses format selected in Preferences
##TIME## Current time - uses format selected in Preferences

If Then Else Example

The If Then Else structure is a little more complicated than the other tags, but is useful for only outputting certain tags or text if a particular condition is met.  For example:

<#IF-IS-GROUP Group ##ELSE Track ##ENDIF#>

In the above example, the text "Group" will be output if the track is a group header, otherwise it will output the string "Track".  See the "list_template" template file to see how the If Then Else structure is used.

Special Tags

Templates may also contain special HTML tags.  These tags handle repeating groups of data, like multiple effects per track and multiple tracks per project.
 

<!-- TRACK_START --> Marks the beginning of the track info
<!-- TRACK_END --> Marks the end of the track info
<!-- FX_START --> Marks the beginning of the effects info
<!-- FX_END --> Marks the end of the effects info
<!-- ITEM_START --> Marks the beginning of the item info
<!-- ITEM_END --> Marks the end of the item info
<!-- SEND_START --> Marks the beginning of the item info
<!-- SEND_END --> Marks the end of the item info
<!-- RECEIVE_START --> Marks the beginning of the item info
<!-- RECEIVE_END --> Marks the end of the item info

With the track start and end tags, you tell ReaperDoc what HTML to use for every track.  So the template defines the track format once, but ReaperDoc will duplicate that track format for every track in the project.

The effects block is marked with the FX start and end tags, and both tags should be within the track tags.

The best way to see how this is used is to look at the default_template.html file.
